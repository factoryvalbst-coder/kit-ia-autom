Documenter un programme automate non commenté avec l'IA

Lecture 4 min · Kit IA de l'Automaticien

Reprise de machine, départ du concepteur, sous-traitant injoignable : tu te retrouves devant un programme sans commentaires, des variables cryptiques, zéro documentation. Avant toute modification, il faut comprendre l'intention. Un LLM lit ce genre de code très vite et t'en génère un premier jet de documentation que tu corriges.

La méthode

  1. Exporte un bloc lisible (Structured Text, liste d'instructions, ladder décrit).
  2. Anonymise (jamais d'identifiant client dans un LLM public).
  3. Lance le prompt : il produit une explication par section + un tableau des variables.
  4. Valide l'intention métier — la seule chose que l'IA ne peut pas inventer.

Le prompt prêt à copier

Prompt — documenter un programmeTu es un automaticien senior qui reprend un programme non documenté. CODE (anonymisé) """ [colle ici le bloc] """ ACTION 1. Résume en 3 lignes ce que fait ce bloc. 2. Découpe-le en sections logiques et explique le rôle de chacune. 3. Génère un TABLEAU des variables : nom | type supposé | rôle probable. 4. Propose des commentaires de code prêts à coller. GARDE-FOU - Distingue ce qui est CERTAIN (lecture du code) de ce qui est SUPPOSÉ (intention métier). Marque tout supposé par [SUPPOSITION].

Pourquoi le marqueur [SUPPOSITION] change tout

Forcer l'IA à séparer le certain du supposé t'évite de prendre une hypothèse pour une vérité. Tu sais exactement quoi aller vérifier sur la machine avant de modifier quoi que ce soit.

Exemple

Sur un bloc de régulation de niveau par pompe avec hystérésis, l'IA restitue : le résumé (« régulation de niveau, hystérésis 20/80 %, coupure sécurité »), le découpage par sections, et un tableau des variables où le pourcentage de niveau est marqué [SUPPOSITION] faute de contexte. Tu valides ou corriges ce point précis, puis tu colles les commentaires générés.

Documentation n'est pas validation

Une doc générée t'aide à comprendre, pas à certifier. Sur une machine en exploitation, confronte toujours l'explication de l'IA au comportement réel avant d'intervenir.

Questions fréquentes

L'IA comprend-elle le Structured Text et le ladder ?
Elle lit très bien le Structured Text et la liste d'instructions. Pour le ladder, fournis-en une description textuelle ou une logique exportée ; elle reconstitue la structure.
Peut-on lui faire commenter directement le programme ?
Oui : demande des commentaires prêts à coller, section par section. Relis-les avant intégration, surtout ceux marqués comme suppositions.
Est-ce risqué pour la confidentialité ?
Oui si tu colles du code client tel quel. Anonymise les noms, repères et adresses réseau, ou utilise un modèle hébergé en EU ou en local pour les données sensibles.
